# NeGO
중고거래 웹 플랫폼 (팀 프로젝트)

---

## 프로젝트 개요

NeGO는 실제 중고거래 서비스의 사용자 흐름을 분석하여  
상품 등록부터 거래, 채팅, 후기, 관리자 기능까지 구현한 중고거래 웹 플랫폼입니다.

단순 CRUD를 넘어  
안전거래, 실시간 채팅, 신고 및 신뢰지수 시스템을 통해  
실제 서비스와 유사한 구조를 목표로 개발했습니다.

---

## 프로젝트 목표

- 실제 중고거래 플랫폼의 사용자 흐름 분석
- 비회원 / 회원 / 관리자 권한 분리
- 거래 안정성을 고려한 서비스 설계
- 백엔드 중심의 웹 서비스 구조 이해 및 구현

---

## 벤치마킹

- 중고나라 : 상품 등록·검색·거래 기본 흐름
- 번개장터 : 안전거래 프로세스
- 당근마켓 : 사용자 신뢰도 및 후기 시스템

기존 서비스들의 핵심 기능을 분석하여  
중고거래 서비스에 필수적인 기능 위주로 구현했습니다.

---

## 핵심 기능

### 비회원 / 회원 공통
- 상품 조회 및 검색
- 시세 조회
- 문의 게시판

### 회원 기능
- 상품 등록 / 수정 / 삭제
- 안전거래 프로세스
- 찜 기능
- 거래 후기 및 신뢰지수 관리
- 마이페이지 (구매·판매·배송·후기 관리)
- 실시간 채팅 (WebSocket)

### 관리자 기능
- 회원 관리
- 상품 및 카테고리 관리
- 신고 관리
- 공지사항 관리

---

## 시스템 설계

- ERD 기반 데이터베이스 설계 (회원, 상품, 거래, 채팅, 신고 등)
- Use Case 및 Sequence Diagram 기반 기능 흐름 설계
- 역할 기반 권한 분리 (비회원 / 회원 / 관리자)

## 담당 기능 (김효준)

본 프로젝트에서 팀장으로서 전체 구조 설계와 핵심 기능 구현을 주도했습니다.

### 서비스 구조 및 설계
- 중고거래 서비스 전반의 사용자 흐름 설계
- 비회원 / 회원 / 관리자 권한 구조 설계
- 주요 도메인(회원, 상품, 거래) 관계 정의
- ERD 기반 데이터베이스 구조 설계 주도

### 마이페이지
- 구매 / 판매 내역 조회
- 거래 상태별 목록 관리 (판매중 / 거래중 / 판매완료)
- 배송 상태 확인
- 후기 작성 및 관리
- 사용자 활동 내역 통합 조회

### 거래 및 안전거래
- 구매자 / 판매자 화면 분리 설계
- 거래 단계별 상태 관리 로직 구현
- 거래 완료 시 상품 상태 자동 변경
- 안전거래 버튼 활성·비활성 처리

### 후기 및 신뢰지수
- 거래 후기 작성 기능 구현
- 후기 기반 신뢰지수 증감 로직 구현
- 신뢰지수에 따른 사용자 상태 관리

### 시세 조회
- 상품명 / 카테고리 기반 시세 조회
- 평균·최대·최소 가격 산출 로직
- Chart.js 기반 시세 그래프 시각화

### 출석 체크
- 일일 출석 체크 기능
- 출석 여부 DB 관리
- 출석 시 신뢰지수 증가 로직

### 관리자 기능
- 회원 관리
- 신고 관리 및 강제 탈퇴 처리
- 카테고리 관리 (상·하위 구조)
- 공지사항 관리

### 협업 및 팀 운영
- 팀 일정 관리 및 역할 분담
- 기능 구현 가이드 제공 및 코드 리뷰
- PPT 및 프로젝트 문서 정리 주도
